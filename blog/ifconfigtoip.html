Switching from ifconfig to ip

About a year ago, I realized ifconfig had been deprecated in favor of iproute2 (ip + tc). I thought "Oh, I should probably get used to using ip." And then, I completely forgot about it, and continued using ifconfig. Only recently did I realize how much I've come accustomed to ifconfig, setting up a laptop without internet, I needed to do some NIC operations in order to get my wifi working again, and realized that I didn't have net-tools installed! Typically I'd apt-get install net-tools and be on my merry way, but of course I couldn't do this, because I didn't have internet. I ended up looking at the manpage to figure out how to do what I needed to in ip, and that insipred me to make this migration guide / cheatsheet for migrating from ifconfig to ip. In each section, the relevant ip command is one line below the ifconfig command. Disclaimer! This guide is far from complete, it's just the commands I find handy.

List all interfaces:
ifconfig -a
ip a

Set interface up/down:
ifconfig <interface> up/down
ip link set <interface> up/down

Change interface address:
ifconfig <interface> address <address>
ip a add <address> dev <interface>

Set interface MAC:
ifconfig <interface> hw ether <MAC>
ip link set dev <interface> address <MAC>

Enable / Disable promisc:
ifconfig <interface> promisc/-promsic
ip link set dev <interface> promisc on/off

Enable / Disable multicast:
ifconfig <interface> multicast/-multicast
ip link set dev <interface> multicast on/off

Setting TX queue length:
ifconfig <interface> txqueuelen <queuelen>
ip link set <interface> txqueuelen <queuelen>
